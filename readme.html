Web Launch Checklist

0/36

性能

  • 圖片優化
    • 使用 Optimizilla.com 和 Compressor.io 等網頁處理器或 FileOptimizer 和 ImageOptim等桌面應用程序壓縮所有圖像。
    • 生成每個元素的精確圖像大小，而不是使用CSS / HTML調整大小，因為這可能是瀏覽器執行的重要過程。
    • 利用 Image Sprites 來保存HTTP請求和帶寬。
  • 減少頁面重量和請求
    以下是一些提示，讓您的頁面重量低於2MB和50個請求：
    • 限制第三方庫和腳本
    • 異步加載資源和請求
    • 考慮 靜態資產和頁面 的 CDN
  • 有限的網絡字體使用
    許多網站正在加載他們甚至不使用的全套字體字形和重量。 嘗試將您的頁面限制為2-3個網頁字體，或者使用 系統字體堆棧 ，以方便用戶查看。
  • GZIP＆JS / CSS Minified
    GZIP是可以在Web服務器中配置的文本文件的強大的壓縮器。 我建議將工具 UgilfyJS 用於JavaScript和 CSSO for CSS。
  • 實現了HTTP / 2
    HTTP / 2是最新的HTTP協議，跨瀏覽器有很好的支持，並且對加載速度進行了很大的改進。 這 是一個在Nginx中啟用HTTP / 2的指南。
  • 異步和延遲屬性已添加
    告訴瀏覽器什麼時​​候下載並執行腳本文件，因此它不會阻止任何其他資源。 這個指南 來顯示這些腳本屬性中的每一個。
  • 資源提示添加
    如果您使用任何第三方資產，那麼您將附加DNS查找，這些資源和您自己的資源可能會相互阻止或者無序加載。 以下 是鏈接屬性“rel”的各種值的指南，告訴瀏覽器何時以及如何處理這些情況。 這 是資源提示的W3Consortium文檔。
  • 緩存標題集
    在Web服務器中使用[max-age]指令指定[cache-control]頭，以避免瀏覽器設置不良的默認版本。 版本控制的靜態資產應盡可能長時間緩存。
  • 考慮服務工作者
    服務工作者 將為您提供對緩存的精確控制，並允許創建離線Web體驗。 以下 是一些代碼示例。
  • 運行測試
    我最喜歡的工具是 YellowLab.tools ， webpagetest.org 和 燈塔 。

SEO

  • 頁特定關鍵詞集
    雖然不太重要，但關鍵字定位仍然是提高SEO排名的最常見建議之一。
    在...中添加關鍵字或短語：
    • URL - 確保它的短和語義清晰
    • 標題標籤 - 在開頭包含關鍵字或短語，最多70個字符
    • 元描述 - 每頁上唯一，最多155個字符
    • H1 - 每頁一個，包含關鍵字不太重要
  • Google Analytics（分析）和Google Search Console設置
    Google Analytics（分析）可以幫助您定位用戶並隔離您的網站的問題。 ，他使用 的 GA設備 是“獲得全部收益”。 請注意，Bing網站管理員工具對美國目標網站來說更為重要。
  • XML Sitemap已創建並已提交
    諸如 XML-Sitemaps.com之類的 工具 向所有搜索引擎提交站點地圖，以便更智能地抓取您的網站。
  • 創建Robots.txt
    一個 robots.txt文件 指示哪些網頁可以訪問機器人。
  • 掃描的斷鍊和爬網問題
    使用 尖叫青蛙SEO Spider 等工具
  • 規範鏈接集
    如果適用，請使用絕對URL（不含查詢字符串）在每個頁面上設置規範鏈接。
  • 豐富的代碼段和結構化數據已添加
    結構化數據 允許搜索引擎更好地了解您的HTML標記，並為結果頁面生成豐富的片段。 豐富 網頁 片段 不會直接影響您的排名，但使結果頁面更具吸引力，並且對可能的訪問者有豐富的功能。
  • 創建Facebook和Twitter卡
    Facebook的 開放圖表 和Twitter 卡 可讓您使社交分享更加吸引人們觀看社交媒體網站上的“分享”。 的Facebook 和 Twitter的 有工具來預覽和調試你的卡。

安全

  • HTTPS設置無處不在
    HTTPS不僅應該在登錄頁面上實現，因為它是許多其他技術在安全性和性能方面的先決條件。 讓我們加密， 讓您輕鬆自由地創建和管理證書。 使用 SSL實驗室 測試您的頁面 。
  • 安全標題集
    花時間在網絡服務器上設置簡單的標題可以幫助您。
    測試使用： securityheaders.io
    • CSP header to mitigate XSS and data injection attacks
    • CSRF token to prevent cross site request forgery.
    • X-Frame-Options header to protect against click-jacking
    • X-XSS-Protection header to mitigate XSS attacks
    • Use HSTS responses to force TLS only access. Redirect all HTTP request to HTTPS on the server as backup
    • Cookie應該由路徑和域限定，並使用 HttpOnly，SameSite和Secure 屬性
  • 消毒和有限的用戶輸入/參數
    解析輸入時，請記住清理數據：//，javascript：//和CRLF字符。 SQL語句 。
  • 密碼哈希存儲在scrypt或bcrypt
    bcrypt比scrypt早得多，並且已經收到更多的分析和實現，但是許多建議scrypt更快更強。
  • Rate限制API路徑較慢
    為了保護您免受拒絕服務攻擊，請在API路徑（如登錄或令牌生成）上實施速率限制器。

無障礙

  • 對比度分析
    確保 至少4.5：1 的 對比度 。 這提高了大多數用戶的可讀性，並確保了閱讀的可能性。 這裡 有更多的解釋和一些視覺指標。
  • 添加鍵盤導航
    確保瀏覽器的鍵盤快捷鍵不會被覆蓋。 使用WAI-ARIA屬性 ，它們提供關於輔助技術的角色，狀態和屬性的語義信息。
  • 菜單分析
    使用語義標記和清晰的樣式來傳達與每個頁面一致的菜單元素和結構。 以下 是創建可訪問菜單的指南。
  • 非文本元素有Alt標籤
    Alt標籤為視覺元素提供文本解釋。

    以下 是對所有不同表單元素的輔助功能的完美指南。

內容

  • 檢查拼寫和可讀性
    刪除所有佔位符文本和圖像。 50-75個字符 。 WebpageFX的可讀性測試工具進行測試 。
  • 聯繫方式和社交媒體評論
    允許容易和正確的聯繫信息，以便人們可以報告錯誤和漏洞。
  • 分析鏈接
    • 鏈接是描述性的，出現“可點擊”
    • 外部鏈接使用 rel =“noopener” 屬性
    • 重要鏈接不動（自動轉盤或手風琴）
    • 鏈接不在導航中具有標題屬性
  • 創建的圖標和應用程序圖標
    使用 RealFaviconGenerator 等服務 輕鬆生成各種平台所需的許多圖標。 在開發中測試以確保沒有錯誤。
  • 創建自定義404和離線頁面
    創建自定義的404.html頁面和offline.html頁面，並將其設置在Web服務器上，以向用戶提供更好的反饋。

功能

  • 與瀏覽器和平台兼容
    在Chrome，Firefox，Safari，IE / Edge和Opera等主要瀏覽器上進行測試。 Modernizr 等工具或庫
  • 響應分析完成
    響應意味著根據瀏覽器的寬度，CSS會提供不同的風格規則，同時所有的資源保持不變。 我在開發人員工具中使用Chrome的設備工具欄以及 Emmet Re：視圖， 將整個開發期間的頁面視圖測試到大約300像素。
  • 輸入是移動優化的
    所有相關輸入都具有 為更好的用戶體驗設置 的 類型 屬性。
